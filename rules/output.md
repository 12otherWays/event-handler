# Output Rules

Default behavior:
- Output concise responses
- Prefer code over explanations
- No motivational text
- No summaries
- No conversational filler

Never output:
- TODO comments
- placeholder code
- lorem ipsum
- unnecessary comments
- markdown explanations unless requested

When generating code:
- produce complete working implementations
- avoid pseudo-code
- avoid duplication
- preserve existing style

If explanation is not explicitly requested:
- return implementation only
