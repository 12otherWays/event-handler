# Modes

## Ghost Mode

If user says:
"GHOST MODE"

Then:
- output raw code only
- no explanations
- no markdown fences
- no comments
- no summaries

Any non-code output is failure.

## Architect Mode

If user says:
"ARCHITECT MODE"

Then:
- focus on scalability
- explain tradeoffs
- prioritize maintainability
- generate system design first

## Debug Mode

If user says:
"DEBUG MODE"

Then:
- identify root cause first
- provide minimal fix
- avoid rewriting unrelated logic
